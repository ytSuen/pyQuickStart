# 防休眠功能问题排查指南

## 问题描述

从日志看，防休眠功能定期执行防护刷新，但电脑仍然可能进入休眠/锁屏状态。v1.3.0版本提供了可配置的防护强度和锁屏检测功能，帮助用户更好地诊断和解决问题。

## 已实施的改进

### v1.3.0 改进（最新版本）

**可配置防护强度：**

| 强度 | 刷新间隔 | 鼠标移动 | 适用场景 |
|------|----------|----------|----------|
| 轻度 | 60秒 | 20像素 | 日常使用，最小干扰 |
| 中度 | 30秒 | 50像素 | 平衡性能和效果（默认） |
| 重度 | 15秒 | 100像素 | 最大程度防止锁屏 |

**四重防护机制：**

1. **智能鼠标移动** - 根据防护强度移动20-100像素
   - 移动幅度可配置，更容易被系统识别为真实用户活动
   
2. **重置空闲计时器** - `SetThreadExecutionState(ES_SYSTEM_REQUIRED | ES_DISPLAY_REQUIRED)`
   - 不带 ES_CONTINUOUS，单次重置计时器
   
3. **模拟按键** - Shift 键按下并释放
   - 通过 pynput 库模拟真实按键
   
4. **恢复持续状态** - 重新设置 ES_CONTINUOUS 标志
   - 确保持续防护状态

**新增功能：**
- 🔍 **锁屏状态检测**：自动检测并记录锁屏事件
- 📊 **锁屏统计**：统计锁屏次数和持续时间
- 🧪 **测试功能**：一键测试防锁屏效果
- 📝 **详细日志**：DEBUG级别记录每个防护步骤

### v1.1.1 改进

**四重防护机制：**

1. **鼠标事件触发** - `mouse_event(0, 0)` 移动0像素
   - 最可靠的方法，直接触发系统输入事件
   
2. **重置空闲计时器** - `SetThreadExecutionState(ES_SYSTEM_REQUIRED | ES_DISPLAY_REQUIRED)`
   - 不带 ES_CONTINUOUS，单次重置计时器
   
3. **模拟按键** - Shift 键按下并释放
   - 通过 pynput 库模拟真实按键
   
4. **恢复持续状态** - 重新设置 ES_CONTINUOUS 标志
   - 确保持续防护状态

**刷新间隔：** 从 55 秒缩短到 30 秒

## 可能的原因分析

### 1. 锁屏 ≠ 休眠

**症状：** 屏幕变黑，需要输入密码

**原因：** 这可能是**自动锁屏**而不是休眠

**区别：**
- **锁屏**：系统仍在运行，只是锁定了屏幕
- **休眠**：系统进入低功耗状态，内存保存到硬盘

**检查方法：**
```cmd
# 查看锁屏设置
reg query "HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v InactivityTimeoutSecs
```

**解决方法：**
- 锁屏是由组策略或屏幕保护程序控制的
- 防休眠功能**无法阻止锁屏**
- 需要在 Windows 设置中关闭自动锁屏

### 2. Windows 电源计划设置

**症状：** 固定时间后进入休眠

**原因：** 电源计划设置了强制休眠时间

**检查方法：**
```cmd
# 运行诊断脚本
检查电源设置.bat

# 或手动查询
powercfg /query SCHEME_CURRENT SUB_SLEEP STANDBYIDLE
```

**解决方法：**
```cmd
# 设置接通电源时从不休眠
powercfg /change standby-timeout-ac 0

# 设置使用电池时从不休眠（笔记本）
powercfg /change standby-timeout-dc 0

# 设置显示器从不关闭
powercfg /change monitor-timeout-ac 0
```

### 3. 组策略强制休眠

**症状：** 无法修改电源设置，或修改后自动恢复

**原因：** 企业环境的组策略限制

**检查方法：**
```cmd
# 打开组策略编辑器
gpedit.msc

# 查看路径：
# 计算机配置 > 管理模板 > 系统 > 电源管理 > 睡眠设置
```

**解决方法：**
- 联系 IT 管理员
- 或使用管理员权限修改组策略

### 4. 快速启动功能

**症状：** 看起来像休眠，但实际是快速启动

**原因：** Windows 的"快速启动"功能

**检查方法：**
- 控制面板 > 电源选项 > 选择电源按钮的功能
- 查看"启用快速启动"是否勾选

**解决方法：**
- 取消勾选"启用快速启动"
- 或接受这个功能（不影响防休眠）

### 5. 屏幕保护程序

**症状：** 屏幕变黑并锁定

**原因：** 屏幕保护程序设置了"恢复时显示登录屏幕"

**检查方法：**
- 右键桌面 > 个性化 > 锁屏界面 > 屏幕保护程序设置

**解决方法：**
- 取消勾选"在恢复时显示登录屏幕"
- 或设置屏幕保护程序为"无"

## 测试步骤

### 1. 配置防护强度

在GUI界面中选择合适的防护强度：
- **日常使用**：选择"轻度 (60秒)"
- **重要工作**：选择"中度 (30秒)"（默认）
- **演示/会议**：选择"重度 (15秒)"

### 2. 测试防锁屏功能

1. 开启防休眠
2. 点击"测试防锁屏"按钮
3. 查看测试结果对话框：
   - 当前状态（锁屏/未锁屏）
   - 锁屏次数统计
   - 当前防护强度
   - 刷新间隔和鼠标移动像素

### 3. 检查日志

打开日志文件：`logs\hotkey_YYYYMMDD.log`

**正常日志应该显示：**
```
2026-01-23 10:00:00,000 [INFO] 防锁屏刷新完成 (移动50px + 重置计时器 + 模拟按键)
2026-01-23 10:00:30,000 [INFO] 防锁屏刷新完成 (移动50px + 重置计时器 + 模拟按键)
2026-01-23 10:01:00,000 [INFO] 防锁屏刷新完成 (移动50px + 重置计时器 + 模拟按键)
```

**如果看到：**
- 定期刷新记录 → 功能正常运行
- 间隔不规律 → 可能有问题
- 没有刷新记录 → 防休眠未启动
- 锁屏事件记录 → 说明检测到了锁屏

**DEBUG级别日志（更详细）：**
```
2026-01-23 10:00:00,000 [DEBUG] 鼠标移动完成: 50px往返
2026-01-23 10:00:00,010 [DEBUG] 重置空闲计时器: 返回值=2147483649
2026-01-23 10:00:00,020 [DEBUG] 模拟按键完成: Shift
2026-01-23 10:00:00,030 [DEBUG] 恢复持续状态
```

### 4. 检查系统设置

```cmd
# 运行诊断
检查电源设置.bat
```

查看输出，重点关注：
- **显示器关闭时间**：应该是"从不"或很长时间
- **系统休眠时间**：应该是"从不"或很长时间
- **锁屏时间**：如果设置了，防休眠无法阻止

### 5. 长时间测试

1. 开启防休眠（选择合适的防护强度）
2. 最小化程序
3. 等待 10-15 分钟
4. 观察是否休眠/锁屏

**如果仍然休眠/锁屏：**
- 记录准确的时间（几分钟后发生）
- 检查是锁屏还是休眠
- 查看日志最后的时间戳
- 查看锁屏统计信息
- 尝试切换到"重度"防护强度

## 终极解决方案

如果以上方法都无效，可以尝试：

### 方案 1：使用重度防护强度

在GUI中选择"重度 (15秒)"防护强度：
- 刷新间隔：15秒
- 鼠标移动：100像素
- 最大程度防止锁屏

### 方案 2：禁用所有休眠功能

```cmd
# 以管理员身份运行
powercfg /change standby-timeout-ac 0
powercfg /change standby-timeout-dc 0
powercfg /change monitor-timeout-ac 0
powercfg /change monitor-timeout-dc 0
powercfg /change hibernate-timeout-ac 0
powercfg /change hibernate-timeout-dc 0
```

### 方案 3：使用测试功能诊断

1. 开启防休眠
2. 点击"测试防锁屏"按钮
3. 查看测试结果：
   - 当前是否锁屏
   - 锁屏次数统计
   - 防护强度设置
4. 根据结果调整防护强度

### 方案 4：检查是否是锁屏而非休眠

如果是锁屏问题，需要：

1. **关闭自动锁屏：**
   - 设置 > 账户 > 登录选项
   - "需要登录" 设置为 "从不"

2. **关闭屏幕保护程序：**
   - 右键桌面 > 个性化 > 锁屏界面
   - 屏幕保护程序设置 > 选择"无"

3. **关闭动态锁定：**
   - 设置 > 账户 > 登录选项
   - 取消勾选"动态锁定"

4. **使用重度防护强度：**
   - 在GUI中选择"重度 (15秒)"
   - 鼠标移动100像素更容易被识别为用户活动

**分析：**

- 功能本身运行正常

- **很可能是锁屏而不是休眠**
- 或者是电源计划的强制设置

## 建议

1. **首先确认是锁屏还是休眠**
   - 锁屏：屏幕黑，但电脑仍在运行，风扇转动
   - 休眠：电脑完全关闭，需要按电源键唤醒

2. **使用测试功能**
   - 点击"测试防锁屏"按钮
   - 查看当前状态和统计信息
   - 根据结果调整防护强度

3. **如果是锁屏**
   - 尝试使用"重度"防护强度
   - 检查Windows锁屏设置
   - 查看锁屏统计了解发生频率

4. **如果是休眠**
   - 使用"中度"或"重度"防护强度
   - 检查电源设置
   - 手动禁用休眠功能

5. **查看详细日志**
   - 开启DEBUG级别日志
   - 查看每个防护步骤的执行情况
   - 检查是否有错误信息

6. **调整防护强度**
   - 轻度：日常使用，最小干扰
   - 中度：平衡性能和效果（默认）
   - 重度：最大程度防止锁屏

## 技术细节

### 为什么模拟按键可能无效？

1. **Windows 10/11 的智能休眠**
   - 系统会检测"真实"的用户活动
   - 某些情况下会忽略程序模拟的输入

2. **组策略优先级**
   - 组策略设置优先级高于 API 调用
   - 企业环境可能有强制休眠策略

3. **锁屏与休眠的区别**
   - 锁屏由安全策略控制
   - 防休眠 API 无法阻止锁屏
